# Report Format

The analyzer writes `DI_AUDIT_REPORT.md`.

Sections:

1. Summary and overall result.
2. Rule coverage.
3. Blocking findings.
4. Warnings.
5. Informational findings.
6. Service registrations.
7. Unresolved dependencies.
8. Input and workspace diagnostics.
9. Ambiguous or skipped analysis.
10. Suggested fix order.

The report separates deterministic blocking findings from heuristic warnings and partial/skipped analysis.
